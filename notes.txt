websocket begins with a Http GET Request from client to server, called the 'handshake'. This request carries with
it a Connection: upgrade, and Upgrade: websocket header to tell the server that it wants to begin a WebSocket connection,
and a Sec-WebSocket-Version header that indicates the kind of response it wants.

The request header also include a Sec-WebSocket-Key field. With this, the server creates a Sec-WebSocket-Accept header that
forms part of its response.



// Client HandShake Request

GET /chat HTTP/1.1
Host: example.com:8000
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13


server:- types :- add, remove, msg, img
client:- types :- alert, msg

{"typ":"add", "userName":"cosmicoppai"}  -> {'typ':'alert', 'msg':'cosmicoppai has joined the chat', 'token':'random-value', 'totalUser':1}
{"typ":'remove', 'userName':'cosmicoppai', 'token':'random-value'} -> {'typ': 'alert', 'msg':'cosmicoppai has left the chat', 'totalUser':0}
{'typ':'message', 'msg':'yo'} -> {'typ':'message', 'msg':'yo', "userName":'cosmicoppai'}
